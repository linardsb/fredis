**Slack Socket Mode silent-drop fix shipped (2026-05-01 ~06:30–06:45 BST)**

- **Root cause:** slack-sdk Socket Mode WebSocket goes half-open silently — process stays alive, no exception, systemd never restarts. This was the second occurrence of the pattern.
- **Fix deployed (commit `2bc4034`, then `44448c2`):** 30-min forced reconnect cycle via `SlackAdapter.reconnect()` in a `_socket_watchdog` asyncio task. Skips cycle if a turn is mid-reply (`note_turn_start/end` from router). 3 consecutive reconnect failures → `sys.exit(1)` → systemd picks it up. Tunable via `CHAT_SLACK_RECONNECT_SEC` (0 disables).
- **Files changed:** `.claude/chat/adapters/slack.py`, `.claude/chat/router.py`, `.claude/chat/main.py`, `.claude/scripts/config.py`.
- **Tests:** 4 new unit tests added; full suite 42/42 green.
- **Deployed to VPS** at 06:44 BST; first forced reconnect expected ~07:14 BST.
- **Decision:** chose "skip reconnect if turn in-flight" approach (option a) over letting HTTP retries absorb mid-recycle failures — explicit and safer.
- **Lesson:** `is_connected()` on slack-sdk Socket Mode returns stale `True` on half-open sockets — can't be trusted as a health check. Periodic forced reconnect is the recommended pattern per Slack docs.
