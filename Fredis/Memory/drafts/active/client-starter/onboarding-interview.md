# Client Second-Brain — Onboarding Interview

**Purpose:** Capture the texture of who the client is, how they work, what they believe, and where they're going — so the brain's memory files produce an assistant that *knows them*, not one that just has their name on file.

This is saulera's client-intake instrument. It is generalised from the original Fredis onboarding (172 questions) and stripped of any one person's answers. Run it per client.

---

## How to run this

**Tier by client type — don't hand everyone all 172 questions:**

| Tier | Sections | Run it for |
|------|----------|-----------|
| **Tier 1 — Core** | A B C D E F G H J K Q AA | **Every client** (owner-run SMB — clinic, trades, agency, accountant, e-commerce) |
| **Tier 2 — Business depth** | I L N O P | Founders, agencies, multi-site or multi-market operators |
| **Tier 3 — Deep personal** | M R S T U V W X Y Z | A full second brain for a founder/operator who wants the lot |

**Depth tags within each question:**
- **★** = core (the brain needs this to function)
- **★★** = rich (real personalisation)
- **★★★** = deep (transformative — optional, but this is where the magic lives)

**Minimum viable:** all ★ in Tier 1.  **Good setup:** all ★ + ★★ in Tier 1 (+ Tier 2 if relevant).  **Great setup:** everything in the tiers that fit the client.

**How to answer:** inline under each question — bullets, paragraphs, stories, whatever fits. Longer is usually better. Blank / `TBD` is fine for skipping. No need to do it in one sitting; the file waits.

## House defaults (changeable per client)

- **Obsidian plugins** — Templater + Dataview
- **Linking style** — wiki-links (`[[Client Name]]`)
- **Folder shape** — flat (until an entity hits 3+ files)
