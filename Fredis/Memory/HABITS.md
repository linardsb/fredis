# Daily Habits - 1% Better Every Day

_Inspired by James Clear's Atomic Habits. The goal is one intentional improvement per day in each pillar._

## Pillars

1. **Ship — Client Delivery**
   One concrete artifact forward per day. A sent draft, a deployed piece, a review delivered, a spec signed off. Not "worked on X" — something moved.
   _Compounds into:_ case studies, referrals, invoices, reputation as someone who ships.

2. **Frontier — AI Agentic Craft**
   One intentional build / experiment / read at the edge of what's actually possible today. Not "read about AI" — built or tested something new.
   _Compounds into:_ the "why hire Linards" differentiation that justifies consulting fees above commodity rates.
   _Energising pillar (I5):_ when stuck, tired, or avoiding Ship work, surface one small Frontier thread (a half-finished `research/` experiment, a sketched idea, a 20-minute build) — getting one Frontier win restarts the engine. **Don't** dangle Frontier on days where Ship is legitimately urgent — that turns it into procrastination bait.

3. **Read — Markets & Systems**
   One deliberate read on commodity / public-sector / UK-LV-AR geopolitical-finance signal. Not doom-scrolling — a chart studied, a report skimmed with a question in mind.
   _Compounds into:_ analytical edge that makes Linards unlike any other AI consultant. Genuine curiosity, not an imposed pillar.

4. **Ground — Body + Near**
   Body and family treated as one pillar because they protect each other.
   _Compounds into:_ the base that lets pillars 1–3 exist in year 5, not just year 1.

_Deliberately left out: a standalone "Learn" pillar (folded into Frontier + Read, tied to application not consumption); a generic "Health/Relationships" (split into Body + Near so neither drowns the other; "easy to do one and call the other done")._

## A Win Today Looks Like

1. **Ship** — Shipped one concrete thing a client can see — a sent reply to a real client email, a merged/deployed piece, a delivered document, or a spec signed off. Not a private branch commit; visible to the counterparty.
2. **Frontier** — Pushed AI agentic craft one step — a commit to a build repo, an agent loop run end-to-end on real data, or a >200-word captured lesson in the daily log. "Read an article" doesn't count unless something was tried.
3. **Read** — Read one market / public-sector / UK-LV-AR signal with intent, and wrote one "so what?" sentence in the daily log. Headline-scrolling is not reading — the sentence is the proof.
4. **Ground (both halves required, not either/or)** —
   - (a) ≥30 minutes deliberate movement — training, walk ≥5k steps, or physical work, AND
   - (b) one genuine touch with someone he loves — ≥5-min call, face-to-face, or a non-logistical message.
   - Both, or the day doesn't count on this pillar.

## Auto-Detection Rules

The heartbeat can auto-check a pillar ONLY when these criteria are met. Routine maintenance does NOT count — it must be a genuine improvement.

**Ship:**
- Auto-check when: Gmail sent to a client domain, HubSpot engagement/ticket closed on a client, Drive external share
- Does NOT count: internal commits, config changes, routine maintenance

**Frontier:**
- Auto-check when: file activity on a build repo + daily log keyword match (agentic, experiment, build, prototype). Fall back to a self-report nudge at 18:00 if no signal
- Does NOT count: dependency updates, reading without trying

**Read:**
- NEVER auto-check. Linards must report this himself — the captured "so what?" sentence in the daily log is the win.

**Ground:**
- Auto-check when: calendar events match movement keywords (training, walk, gym, etc.) AND a named-person event or message log shows a real touch. Both halves required.
- NEVER auto-check the family/Near half from email or message metadata alone — the meaning matters more than the metadata.

The agent surfaces any pillar with no signal by 18:00; silent if all four hit.

## Late-Day Nudge

**Off.** Linards explicitly opted out of late-day pillar nudges. Don't ping at 18:00 about unchecked pillars.

(Exception: Frontier auto-detection's 18:00 self-report fallback above is fine — it's a single ask, not a nudge cycle.)
