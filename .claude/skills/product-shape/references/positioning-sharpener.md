# Positioning Sharpener

> Phase 5.2 skeleton — structural framework + source list. Deep framework bodies to be filled in a follow-up authoring pass.

## Purpose

One sentence + five Dunford bullets that name: best-fit customer, alternative solutions, unique attributes, true value, and the market category. Chris-Lori voice rules enforced — no "we empower X to unlock Y" fluff.

## Frameworks applied (sources for follow-up authoring)

- **April Dunford, *Obviously Awesome*** (2019) — five building blocks of positioning; alternatives first, attributes second, value third.
- **Geoffrey Moore, *Crossing the Chasm*** (1991) — positioning statement template ("For [customer] who [need], [product] is a [category] that [benefit], unlike [alternative], our product [differentiator].").
- **Chris-Lori voice rules** — evidence-first, verdict-direct, invalidator up front. See `_shared/chris-lori-voice.md`.

## Lane weak-starting-position catalog

Each lane starts with a weak default that needs sharpening:

- **Email Hub** — "Workflow tool for ESPs." Too generic; every ESP vendor claims "workflow." Needs: who specifically within an ESP, what narrow workflow.
- **VTV** — "AI for transport." Weak — every transport RFP gets this. Needs: which decision, which operator stage, which measurable outcome.
- **Cab** — "Bolt replacement in Riga." Weakest — competes directly on the strongest competitor's strongest territory. Needs: a named segment Bolt under-serves (drivers, specific geographic corridor, commission structure, LV-language interface, etc.).

## Structure (to be filled)

1. **Lane pre-load + weak-position prompt** — load `_shared/lanes.md`, state the weak default.
2. **Named-competitor teardown** — 3–5 named competitors + their positioning statements (not inferred — actual ones from their sites).
3. **Dunford five blocks** — each block evidence-first.
4. **Devil's-advocate pass** — "what's the strongest case *against* this positioning?"
5. **Chris-Lori voice gate** — rewrite in Lori rhythm (setup / trigger / invalidator / verdict).
6. **One-tweet test** — 280 chars including the category and the differentiator.
7. **Atis £1k gate** — would Atis bet £1k that a real target customer would repeat this back correctly?
8. **Hand-off** — value flows to `pricing-shaper` revision; unique attributes flow to `mvp-architect`.
