№ 01 — THE PROBLEM

The tools you have don't think with you.

three failure modes

Inboxes are silos

Gmail, Slack, Calendar, HubSpot, GitHub, Drive, Sheets — every tool stores a fragment of context. None of them remember the through-line of what you're actually doing this quarter.

Generic AI auto-sends

Off-the-shelf assistants happily ship public messages on your behalf. That's fine for tickets — fatal for client trust, partner agreements, and anything with your face on it. The author has to remain the human.

Memory dies on logout

A chat that forgets last week's decision is a stranger you have to onboard daily. Real leverage starts when the assistant remembers the deal, the constraint, the family plan, the kill-criteria — and uses them.

№ 02 — THE STANCE

Built around three non-negotiables.

manifesto, abridged

Fredis is not a product. It is a one-person operating system, written and re-written in public, that earned its place by getting one person — Linards — through more decisions per day with less friction and zero unauthorised sends.

The first rule is advisor-mode is the boundary. Every external action — replies, posts, ticket moves, calendar invites — lands as a draft in drafts/active/ and a HubSpot review ticket. Linards reviews. Linards sends. The system never speaks in his name.

The second rule is memory is a vault, not a vector. The brain lives as plain Markdown in an Obsidian vault — SOUL, USER, MEMORY, HABITS, HEARTBEAT — versioned in git, synced across machines, readable without the AI. If the model goes away tomorrow, the brain stays.

The third rule is skills over scripts. Twenty-four discrete skills carry the playbooks: a CISO, a CTO, a product manager, a launch-governor, a meeting scribe. Each one is an executable persona — not a wrapper, but an opinion encoded in markdown that knows when to load itself.

Underneath, a heartbeat ticks every fifteen minutes. It scans the vault, watches the gates, drafts what's due, and posts a single line to Slack: "Here's what I'd do if I were you. You decide."

№ 03 — UNIQUE FEATURES

Eight things nothing else on the shelf does.

spec sheet
F-01 / FLAGSHIP● live

The 15-minute heartbeat.

A self-running tick that pulls fresh inbox + calendar + CRM context, scans launch-governance gates, drafts the replies that are overdue, and updates today's daily log. Quiet between ticks. Useful at every tick. You can read what it did at any time — it leaves a trail in daily/YYYY-MM-DD.md.

F-02 / TRUST MODELadvisor

Never-send by design.

The architecture itself enforces advisor-only behaviour. Outbound channels are write-locked at the integration layer. Drafts go to disk, a HubSpot review ticket appears, Slack posts a [DRAFT] notice. The keys to actually send live with the human, not the agent.

F-03 / VOICEmatched

Voice-matched drafts.

Every reply retrieves three structurally similar past sends from drafts/sent/ and copies cadence, hedge-level, sign-off, line breaks. The drafts read like Linards because they were trained on Linards.

F-04 / KILL GATESYAML

Launch-governance gates.

Per-lane YAML kill-criteria (Email Hub, VTV, Cab) read each tick. If a metric crosses, the heartbeat drafts the post-mortem and the pivot memo before the founder finds time to feel attached.

F-05 / VAULTportable

Markdown-first memory.

Five top-level files (SOUL, USER, MEMORY, HABITS, HEARTBEAT) plus a daily log directory. Versioned in git, opened in Obsidian, readable without the AI. The brain outlives the model.

F-06 / SKILLS24 specialists

An advisory bench, on tap.

Solo-founder, startup-CTO, product-manager, CISO, business-cycle-analyst, IP-overhang-guard, scenario-war-room — each a skill that auto-loads on trigger phrases and answers in its own voice with its own framework. Same brain, different chair.

F-07 / REVIEW QUEUEHubSpot

One queue, five stages.

Every draft creates a HubSpot ticket — Drafted → In review → Needs send → Actioned / Rejected. The queue is the inbox; the file is the content. Approve from a phone, action from a laptop, never lose a thread.

F-08 / IP-AWAREUK · LV

Knows where the legal lines are — and stops you crossing them.

The ip-overhang-guard skill encodes UK CDPA s.11(2), Patents Act s.39, clean-room rebuild triggers and a Merkle-letter template — fires automatically when product work touches anything built on employer time. The uk-latvia-context skill carries the cross-border tax / Companies House / VID admin map. Same brain, dual-jurisdiction citizen.

F-09 / OPENself-hosted

Yours. Not a SaaS.

Runs on a Hetzner VPS, MCP streamable-HTTP transport, BYO model keys. No tenant boundary to leak across. No vendor lock-in to pay rent on.

№ 07 — A DAY WITH FREDIS

The heartbeat, drawn out.

tick = 15 min
06:00 12:00 22:00 heartbeat ticks · q15m
a representative day
  1. 07:30 Wakeup tick.  Pulls overnight Slack DMs, unread Gmail threads, GitHub PR mentions, calendar conflicts for the day. Cross-references the inbox against today's three priority lanes. Drafts the replies that need to land before 9am — voice-matched against three past sends each. Posts a single [DRAFT] notice to #hubspot with the queue link, and creates native Gmail drafts so they're visible the moment Linards opens his inbox. Output: 3–5 reviewable drafts, zero sends.
  2. 10:15 Gate scan.  Heartbeat reads Fredis/Memory/gates/*.yaml — one file per active lane (Email Hub, VTV, Cab). VTV's "qualified bids per week" gate breaches: 0 against a floor of 2. launch-governance auto-drafts a Klein pre-mortem to drafts/active/metrics-gate/ with three branches — pivot the channel, persevere with a smaller test, or trigger the kill criterion. A HubSpot ticket is created in the Needs send stage. Output: a 1-page bet review on Linards's desk before lunch.
  3. 14:00 Meeting capture.  Atis call ends. Linards drops a note: "capture meeting — Atis pricing". The meeting-notes skill structures attendees, agenda, decisions, and action items with owners + ISO dates. Writes to Fredis/Memory/meetings/2026-04-27_atis-pricing.md, then client-log appends a dated entry to retainers/atis.md with the open items. Output: next week's heartbeat already knows what was promised, by whom, by when.
  4. 17:45 EOD synthesis.  memory_synthesis.py rolls up the day: every tool call, every draft created, every decision made or deferred. Writes the structured daily log to Fredis/Memory/daily/2026-04-27.md. memory_reflect.py scans for reusable lessons — patterns that recurred, surprising user feedback, decisions worth remembering — and surfaces MEMORY.md candidates as draft entries for review. Decisions are graded against the morning plan: shipped, slipped, killed. Output: tomorrow you can read why today did what it did.
  5. 22:00 Vault sync.  VPS-side git-sync stages today's diff — daily log, new drafts, MEMORY candidates, gate state changes — and commits with a timestamped message (vault: sync from ubuntu-4gb-hel1-1 @ 2026-04-27T22:00:00Z). Pushes to the private repo. Linards's laptop pulls on next Obsidian open. The brain is now the same on every machine, every device, and durably backed up. Output: tomorrow's heartbeat starts with full context — no warm-up, no re-onboarding.
